# Access Control (QR Codes) - Postman / API Testing

## Setup

Recommended environment variables:

- `BASE_URL` (example: `http://localhost:3000`)
- `ACCESS_TOKEN` (JWT)
- `UNIT_ID` (a unit the user has access to)
- `QR_ID` (DB id of an `AccessQRCode`)

All requests below require:

- `Authorization: Bearer {{ACCESS_TOKEN}}`

Local testing without HikCentral:

- Set `HIKCENTRAL_MOCK=true` (or leave `HIKCENTRAL_BASE_URL` unset).
- Ensure the test user has ACTIVE `UnitAccess` for `UNIT_ID` and `canGenerateQR=true`.

## 1) Generate QR

`POST {{BASE_URL}}/access-qrcodes`

Body:
```json
{
  "unitId": "{{UNIT_ID}}",
  "type": "VISITOR",
  "visitorName": "Ahmed Visitor",
  "validFrom": "2026-02-03T10:00:00.000Z",
  "validTo": "2026-02-03T11:00:00.000Z",
  "permissions": ["ENTER"],
  "gates": ["GATE_A"],
  "notes": "Guest coming for a visit"
}
```

Expected:
- 201/200 with:
  - `qrCode` (DB row)
  - `qrImageBase64`

Test cases:
- `SELF` with no dates -> defaults to 5 minutes.
- `DELIVERY` with no permissions -> defaults to `["DELIVER"]`.
- `VISITOR` without `visitorName` -> 400.
- `validTo <= validFrom` -> 400.
- No ACTIVE `UnitAccess` on `unitId` -> 403.
- `canGenerateQR=false` -> 403.
- Duplicate `(unitId,type)` when `QR_ENFORCE_SINGLE_ACTIVE` is enabled -> 409.

## 2) List QRs (mine)

`GET {{BASE_URL}}/access-qrcodes`

Expected:
- Array of ACTIVE QR rows generated by the requester.

Include inactive:

`GET {{BASE_URL}}/access-qrcodes?includeInactive=true`

Test case:
- Verify that any ACTIVE rows with `validTo <= now` are updated to `EXPIRED` during listing.

## 3) List QRs (by unit)

`GET {{BASE_URL}}/access-qrcodes?unitId={{UNIT_ID}}`

Expected scoping:

- If requester is OWNER for the unit -> returns unit QRs (ACTIVE by default).
- If requester is not OWNER -> returns only QRs they generated for the unit.

Include inactive:

`GET {{BASE_URL}}/access-qrcodes?unitId={{UNIT_ID}}&includeInactive=true`

## 4) Revoke QR

`PATCH {{BASE_URL}}/access-qrcodes/{{QR_ID}}/revoke`

Expected:
- 200 updated `AccessQRCode` row with `status="REVOKED"`.

Test cases:
- Generator revokes their own QR -> 200.
- Unit owner revokes another user's QR for the unit -> 200.
- Non-owner/non-generator -> 403.
- Revoking a non-ACTIVE QR -> 400.
